## 现状 vs Claude Code 记忆系统对比

| 功能 | Claude Code | OpenCode 当前状态 |
|------|-------------|------------------|
| **指令文件自动加载** | CLAUDE.md 多层级加载 | ✅ `instructions` 加载 MANAGEMENT.md |
| **记忆文件自动加载** | MEMORY.md 前200行/25KB | ❌ 未加载 ACTIVE-CONTEXT.md 和 MEMORY.md |
| **Auto memory（自动记忆）** | 自主判断→自动写入 | ❌ 无，全靠手动 |
| **`/memory` 命令** | 浏览/编辑所有记忆 | ❌ 无 |
| **`/remember` 命令** | 快速记录一条信息 | ❌ 无 |
| **记忆位置** | `~/.claude/projects/` 项目外 | `docs/memory/` 项目内 |
| **子代理记忆** | 可独立维护 | ❌ 子代理无记忆 |
| **大小预算** | MEMORY.md ≤200行/25KB | 在 MANAGEMENT.md 有定义但未强制 |

**核心差距总结**：架构层已具备（5层文件系统），但**缺少 auto memory 的自动写入行为** 和 **自动加载机制**。

## 实施计划

### 阶段 1：记忆文件自动加载

```json

  "docs/agent-workflow.md",
  "docs/memory/MANAGEMENT.md",   // 规则
  "docs/memory/MEMORY.md",       // 长期记忆（等价 Claude Code MEMORY.md）
  "docs/memory/ACTIVE-CONTEXT.md" // 热记忆（当前状态）
]
```
