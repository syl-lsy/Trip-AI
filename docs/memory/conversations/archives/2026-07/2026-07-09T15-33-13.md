## 分析

目前的问题在于：

1. **追加全部内容**：只做了 `tail | head` 截断，但大量重复/问候对话仍然被追加
2. **单文件累积**：所有同月的归档都追加到同一个 `summary.md`，没有上限

## 优化方案对比

| 方案 | 做法 | 优缺点 |
|------|------|--------|
| **A: 按文件独立归档** | 改为 `archives/YYYY-MM/文件名.md`，每个对话独立成文件 | ✅ 无单文件膨胀，结构清晰<br/>⚠️ 文件数量可能增多 |
| **B: 提取关键信息 + 限制行数** | 用 grep 只提取含决策/架构/偏好/待办的段落，限制 summary 上限 500 行 | ✅ 文件精炼<br/>❌ 提取逻辑复杂，bash 实现困难 |
| **C: 混合方案** | 按文件独立归档 + 每月首次归档时生成一份摘要索引 | ✅ 兼顾可读性和文件大小<br/>✅ 实现简单 |

### 我推荐方案 A

改为每个已归档对话独立存储文件，`summary.md` 只作为索引：

```
archives/
├── 2026-07-summary.md    # 只包含索引（文件名列表 + 关键决策摘要）
├── 2026-07-08T15-05-05.md
├── 2026-07-08T15-24-49.md
├── 2026-07-09T14-36-40.md
...
```

同时补充 GC 规则：`archives/` 中文件超过 100 个时，清理最旧的。
