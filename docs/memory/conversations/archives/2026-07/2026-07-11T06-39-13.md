## Spec Kit + OpenCode 集成

**Spec Kit** (`github/spec-kit`) 已经内置了 OpenCode 集成（`src/specify_cli/integrations/opencode/__init__.py`）。

### 安装步骤

**1. 安装 Specify CLI**（需要 [`uv`](https://docs.astral.sh/uv/)）：

```bash
uv tool install specify-cli --from git+https://github.com/github/spec-kit.git@v0.12.11
```

**2. 在当前项目中使用 OpenCode 集成初始化：**

```bash
specify init . --integration opencode
```

这将把以下内容安装到 `.opencode/commands/` 中：

| 命令文件 | 用途 |
|---|---|
| `speckit.constitution.md` | 创建项目治理原则 |
| `speckit.specify.md` | 定义构建内容（需求） |
| `speckit.plan.md` | 使用所选技术栈创建实施方案 |
| `speckit.tasks.md` | 生成操作任务列表 |
| `speckit.implement.md` | 根据计划执行所有任务 |

### 工作原理

- 命令以 Markdown 形式存储在 `.opencode/commands/` 中
- `build_exec_args` 将 `/speckit.xxx` 命令映射为 `opencode run --command speckit.xxx <args>`
- 执行以 JSON 格式输出（`--format json`）
- OpenCode 是一个基于 CLI 的工具（`requires_cli: True`），因此 `specify init` 会检查 `opencode` CLI 是否可用
