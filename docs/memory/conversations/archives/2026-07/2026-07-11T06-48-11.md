## OpenSpec 使用流程

你的项目已安装 **openspec CLI v1.6.0**，使用 `spec-driven` schema。它是一套**结构化变更管理**工具，核心工作流如下：

### 完整生命周期（5 步）

```
探索 → 提议 → 实施 → 更新 → 归档
```

| 步骤 | 命令/技能 | 产出 |
|------|----------|------|
| **1. 探索** | `/opsx-explore` | 分析问题、对比方案、理清需求 |
| **2. 提议** | `/opsx-new` | `proposal.md` + `design.md` + `tasks.md` |
| **3. 实施** | `/opsx-apply` | 按 tasks.md 逐步实现代码 |
| **4. 更新** | `/opsx-update` | 变更过程中修订设计/任务 |
| **5. 归档** | `/opsx-archive` | 增量 spec → 主 spec，移入 archive |

### 核心概念

- **Change（变更）**：一次功能的完整生命周期，从提议到归档
- **Artifacts（制品）**：每个 change 下的产物 — `proposal.md`（为什么做）、`design.md`（怎么做）、`tasks.md`（怎么做）、`specs/`（细粒度需求）
- **Delta Spec**：change 中对主 spec 的增量修改，归档时自动合并到主 spec

### 常用命令

| 命令 | 用途 |
|------|------|
| `openspec new change "<name>"` | 创建新变更 |
| `openspec list` | 列出所有变更 |
| `openspec status --change "<name>"` | 查看变更状态 |
| `openspec instructions <artifact> --change "<name>"` | 获取制品模板 |
