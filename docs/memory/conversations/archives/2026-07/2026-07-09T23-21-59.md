# AI Agent 开发规范详解

本规范覆盖大模型 Agent 从设计、开发、测试到运维的全流程，适用于单工具调用 Agent、多步推理 Agent、多 Agent 协作系统等场景，兼容 LangChain/LangGraph、自定义编排、MCP/Skill 生态等主流开发模式，核心目标是保障 Agent **行为可控、结果可测、故障可追溯、能力可迭代**。

## 一、核心设计原则

所有 Agent 开发必须遵守以下底层原则，从源头避免不可控、不可维护的问题。

1. **边界清晰原则**：明确定义 Agent 的能力范围、适用场景和禁止事项，禁止开发无边界的「通用全能 Agent」。
2. **可控可预测原则**：输出结构化、执行有终止条件、行为符合预期，禁止出现无限循环、无意义反复调用。
3. **容错降级原则**：异常场景有明确兜底机制，不崩溃、不静默失败，优先保障基础可用性。
4. **人机协同原则**：高危操作、模糊指令、超出能力边界的场景，主动发起人工确认，不强行自动化。
5. **模块化可复用原则**：能力拆分为可插拔组件（工具、记忆、提示词），避免硬编码耦合，支持快速迭代。
6. **数据最小化原则**：送入大模型的信息仅保留必要内容，敏感数据先脱敏再处理。

## 二、Agent 架构分层规范

严格遵循四层单向依赖架构，职责单一，禁止跨层调用，便于维护和扩展。
