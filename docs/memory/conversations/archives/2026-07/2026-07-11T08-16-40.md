## 错误 1：`import type` 导致 `tool` 无法作为值使用（6 处）

**根因** — 第 1 行用了 `import type { Plugin, tool }`，但 `tool` 是运行时函数（工具创建器），需要作为值导入。`import type` 表示所有导入的符号都仅作类型用，无法在值位置使用。

```ts
// ❌ 错误
import type { Plugin, tool } from "@opencode-ai/plugin"
//                   ~~~~ TS1361: 'tool' cannot be used as a value
```

**修复** — 将 `Plugin` 和 `tool` 分开导入：

```ts
import { type Plugin, tool } from "@opencode-ai/plugin"
```

这样 `Plugin` 仅作类型用，`tool` 作为值导入，可在 `tool({...})` 和 `tool.schema.xxx()` 中使用。

## 错误 2：模块解析设置（1 处）

`TS2307: Cannot find module '@opencode-ai/plugin'` — `tsconfig.json` 缺少 `moduleResolution` 设置。修复 `.opencode/tsconfig.json`：

```json
{
  "compilerOptions": {
    "types": ["node"],
    "moduleResolution": "bundler"
  }
}
```
