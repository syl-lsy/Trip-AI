## 2. Claude Code Auto Memory 的实现机制

Claude Code 的 auto memory 是 **内建于运行时的系统级能力**，不是靠 prompt 实现的：

```
用户行为 → Claude Code 运行时 → 判断"是否需要记住" → 写入 ~/.claude/projects/<project>/memory/
                                                                        ├── MEMORY.md（索引，自动加载）
                                                                        ├── debugging.md（按需）
                                                                        └── api-conventions.md（按需）
```

关键细节：
- 它在**每次模型回复后**由运行时判断，不是模型自主决定
- 判断依据：这条信息在**未来的对话中是否有用**
- 写入时 UI 显示 "Writing memory"，读取时显示 "Recalled memory"
- 开关：`/memory` 命令或 `autoMemoryEnabled: false`

**OpenCode 没有这个内建机制**，无法做到完全一致。最接近的方案是靠 **prompt 指令 + 严格的规则约定**，在每轮回复后让 agent 主动检查是否需要写记忆。缺点是依赖模型遵循指令的可靠性（LLM 不是 100% 遵守规则），但通过充分测试可以达到实用水平。
